# VRCaster 设计文档

> 一键把视频压成 VRChat 兼容版并上传到自己的服务器，产出可直接播放的带 key URL。
> 把手动 ffmpeg 流程（HDR→SDR、无 B 帧、字幕硬编、断点续传、nginx key 鉴权）产品化。

## 1. 技术栈
- **Tauri 2.x**：Rust 后端 + Svelte(TS) 前端
- 平台：Windows 优先（Tauri 跨平台保留）
- ffmpeg/ffprobe：首次运行自动下载到 app data 目录
- 上传：SFTP/SSH（`ssh2` crate）
- 凭据：Windows 凭据管理器（`keyring` crate）

## 2. 核心兼容预设（锁死，用户不可破坏）
VRChat (AMD AVPro) 流畅的关键底座：
- 视频：H.264 **main** profile，**无 B 帧**（`bframes=0`），`yuv420p`
- 容器：MP4 + **faststart**（moov 前置，可边下边播）
- 音频：AAC **立体声**
- HDR：自动检测 → SDR tonemap

## 3. 用户可调项
| 项 | 选项 | 默认 |
|---|---|---|
| 分辨率 | 720p / 1080p / 1440p / 4K（4K 标注"部分客户端可能卡"） | 1440p |
| 码率控制 | 目标大小（填 GB 反算） / 质量档（CRF 高中低） | 目标大小 |
| 编码速度 | 快(fast) / 平衡(medium) / 质量(slow) | 平衡 |
| GPU 硬解 | 自动检测开启 | 自动 |
| HDR tonemap | 自动检测；算法 hable/mobius/reinhard 可选 | hable |
| 音轨 | 自动列出供选，降混立体声，码率 128/192/256k | 192k |
| 字幕 | 内嵌轨选择 / 外挂 .ass/.srt / 关闭；自动提取内嵌字体 + 系统 fallback | 内嵌默认轨 |

## 4. 样片标定（压全片前的试压 + 反推）
1. 选好画质参数后点「压样片」
2. 时间段可选，**默认正中间 2 分钟**（起点 = 时长/2 − 1min，时长 2min 可调）
3. 用**与全片完全相同的滤镜链与编码设定**压样片
4. 完成后展示：实测大小 / 平均码率 / 压制耗时 / 内置播放器预览
5. **反推全片**：
   - 质量档(CRF)：样片实测码率 R → 全片预估大小 = R × (全片时长/样片时长)
   - 目标大小：用样片实测复杂度反推"不超目标大小又尽量保画质"的最优码率
   - 智能建议：基于样片给出"画质↔大小↔耗时"平衡的全片参数推荐
6. 满意后一键压全片

## 5. 批量队列（三种并发模式）
- **顺序流水线（默认）**：压1 →〔传1‖压2〕→〔传2‖压3〕，压制串行满载算力、上传与下个压制重叠满载带宽
- **同步处理 + 顺序上传**（UI 标注：同时压会抢算力，通常不更快）
- **同步处理 + 同步上传**（UI 标注：跨境多流突破不了总带宽）
- 拖入多文件可调顺序；每个文件独立显示分析/参数/双进度

## 6. 服务器与 nginx 助手
- **多 profile**：可存多个 VPS，上传时切换
- 配置项：host / port / user / 认证(密码或私钥) / 远程目录 / URL 前缀 / key
- **凭据安全**：密码、私钥口令存 Windows 凭据管理器，配置文件不含明文
- **nginx 一键部署助手**：生成带 `?key=` 鉴权的 nginx server 块，通过 SSH 自动（装 nginx → 写配置 → `nginx -t` → reload）
- **key**：可一键生成随机 key，部署时同步写进 nginx 与本地 profile

## 7. 上传（吸取教训）
- SFTP 断点续传：首次 `put`，中断后按远程已有大小 `seek` 续传（避开 `reput` 对不存在文件失败的坑），自动重试直到本地/远程大小一致
- 上传前预检远程磁盘空间
- 完成产出 URL（`URL前缀/文件名?key=xxx`），一键复制 + 可选打开浏览器

## 8. UI（深色现代 + 毛玻璃渐变）
- 主界面：大号拖放区 + 队列卡片列表 + 模式切换 + 全局「开始」
- 卡片：源信息标签(4K/HDR/音轨/字幕)、可调参数、「压样片」「压全片」、双进度条、完成态 URL
- 设置页：服务器 profile 管理、nginx 助手、编码默认、语言(中/英)切换
- 紫青点缀渐变、微动效、流畅过渡
- i18n：中英双语，默认中文

## 9. Rust 后端模块划分
| 模块 | 职责 |
|---|---|
| `media.rs` | ffprobe 分析：HDR/色彩、音轨列表、字幕轨列表、时长、分辨率、帧率 |
| `encode.rs` | 构建 ffmpeg 命令（含 tonemap/scale/字幕滤镜）、执行、进度解析、样片模式 |
| `subtitle.rs` | 字幕轨提取、内嵌字体提取、系统字体 fallback |
| `upload.rs` | SFTP 上传 + 断点续传 + 远程空间预检 |
| `server.rs` | nginx 配置生成 + SSH 一键部署 |
| `config.rs` | profile 管理 + keyring 凭据存取 + 配置导出/导入 |
| `queue.rs` | 队列调度（三种并发模式：流水线 / 同步处理 / 同步上传） |
| `ffmpeg.rs` | 首次检测 + 下载 ffmpeg/ffprobe 到 app data |

## 10. 错误处理
ffmpeg 下载失败（可手动指定路径）、源分析失败、压制报错（显示 ffmpeg 日志尾部）、SSH/上传中断（自动续传）、远程空间不足预检。

## 11. 可分发
- Tauri 打包 `.msi`/`.exe`
- 配置可导出/导入（仅非敏感设置，密钥留各自系统密钥环）
- 别人导入后填自己的服务器即可
